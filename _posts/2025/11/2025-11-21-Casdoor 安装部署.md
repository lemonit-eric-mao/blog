# Casdoor 安装部署

## Casdoor是如何自我管理的？

首次运行Casdoor时，会创建一些内置对象以方便其管理：

- 一个内置的命名为 `built-in` 的组织。
- `built-in` 组织下用户名为 `admin`的用户 。
- 一个名为`app-built-in`的内置应用，由`built-in`组织管理，代表Casdoor本身。

`built-in`组织下的所有用户，包括`admin`，都将在Casdoor平台上拥有完全的管理员权限。 因此，如果有多个管理员，建议在`built-in`组织下创建新账户。 或者，应关闭`app-built-in`应用程序的注册通道，以防止不必要的访问。

> #### 注意事项
>
> 无法通过网页用户界面或RESTful API重命名或删除内置对象。
>
> 8Casdoor在许多地方硬编码了这些保留名称；尝试通过修改数据库来重命名或删除它们可能会导致整个系统崩溃。

> Casdoor 支持MySQL、MSSQL、SQLite3和PostgreSQL。 默认情况下，Casto使用MySQL。

> Casdoor将会把users，nodes和topics信息存储在一个名为`casdoor`的MySQL数据库中。

#### caddy/Caddyfile

``` json
casdoor-caddy.com {
        tls internal
        reverse_proxy casdoor:8000 {
                header_up Host {host}
                header_up X-Real-IP {remote}
                header_up X-Forwarded-For {remote}
                header_up X-Forwarded-Proto {scheme}
        }
}

```

- `tls internal` → 使用 Caddy 内置自签证书，不依赖 Let's Encrypt。
- `reverse_proxy casdoor:8000` → 将请求转发到你的服务容器。
- headers → 保留原始请求信息（推荐做法）。



#### 代码服务器`选择原则`

|      | 服务器      | 场景                             | 优势                                                         |
| ---- | ----------- | -------------------------------- | ------------------------------------------------------------ |
| 1    | **Caddy**   | 内网小型部署、快速上线、少量服务 | 配置极其简单，几行 Caddyfile 就能开 HTTPS<br/><br/>内网快速部署，适合小团队快速上线<br/><br/>自动管理 TLS、内置自签证书功能 |
| 2    | **Traefik** | 容器化、微服务、多服务动态管理   | 内网自签证书也能支持<br/><br/>生产环境稳定、易于扩展         |



#### docker-compose.yaml

``` yaml
services:

  caddy:					 # caddy会生产部署的时候一定是要使用HTTPS的
    image: caddy:2-alpine
    container_name: caddy
    restart: unless-stopped  # 容器退出时自动重启，除非手动停止
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./caddy/data:/data       # 容器 /data 映射到相对路径 ./caddy/data，用于存储证书和 TLS 数据
      - ./caddy/config:/config   # 容器 /config 映射到相对路径 ./caddy/config，用于存储 Caddy 配置和状态
      - ./caddy/Caddyfile:/etc/caddy/Caddyfile # 挂载本地 Caddyfile 配置文件到容器
    networks:
      - casdoor-network  # 加入自定义网络，方便 Caddy 与 Casdoor 通信
      
    depends_on:
      casdoor:
        condition: service_healthy  # 等待 Casdoor 健康后再启动 Caddy

  casdoor:
    image: casbin/casdoor:v2.151.0
    container_name: casdoor
    restart: unless-stopped
    ports:
      - "8000:8000"
    environment:
      - GIN_MODE=release  # 设置 Gin 框架为 release 模式（生产模式）
      # 如果你要让 Casdoor 使用 MySQL有两种配置方式
      # 1. 在app.conf文件中修改以下环境变量：
      # 2. 请添加以下环境变量：
      - driverName=mysql
      - dataSourceName=casdoor:casdoorpass@tcp(mysql:3306)/
      - dbName=casdoor
    volumes:
      - ./conf:/conf
      - ./logs:/logs
    networks:
      - casdoor-network

    # 给 Casdoor 加上健康检查
    # 用来 检测Casdoor容器内服务是否可用
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000"]  # 检查服务是否健康，Casdoor 默认 8000 端口
      interval: 5s       # 每 5 秒检查一次
      timeout: 10s       # 超过 10 秒判定失败
      retries: 3         # 连续失败 3 次才判定容器不健康
      start_period: 10s  # 容器启动后等待 10 秒再开始检查

    depends_on:
      mysql:
        condition: service_healthy

  mysql:
    image: mysql:8.4
    container_name: casdoor-mysql
    restart: unless-stopped
    environment:
      MYSQL_ROOT_PASSWORD: password
      MYSQL_DATABASE: casdoor
      MYSQL_USER: casdoor
      MYSQL_PASSWORD: casdoorpass
    volumes:
      - ./mysql-data:/var/lib/mysql
    ports:
      - "3306:3306"
    networks:
      - casdoor-network

    # 给 MySql 加上健康检查
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 5s
      timeout: 3s
      retries: 5
      start_period: 10s

networks:
  casdoor-network:
    driver: bridge

```
